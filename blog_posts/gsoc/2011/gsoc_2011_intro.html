Parrot Foundation is proud to be a part of Google Summer of Code 2011! Parrot was awarded 7 slots this year, which is awesome, considering this is the first year Parrot Foundation is a separate organization. In previous years, Parrot Foundation and The Perl Foundation were both a part of GSoC under a single mega-umbrella organization. This year, after experiencing being the org admin for the mega-umbrella organization for two years, I felt that they could stand on their own feet. Florian Ragwitz dutifully volunteered to herd cats for The Perl Foundation at just the right time, so I wrote the org application and then handed it off to him. All of the awesome stuff you see relating to TPF and GSoC 2011 is a result of his hard work. Without further delay, here are the awesome projects that will be coding in the GSoC spotlight this summer: Title: New Parrot Debugger Student: Kevin Polulak Mentor: Cristoph Otto This is a sorely needed project. I wrote many tests for the Parrot debugger, but due to design decisions before I arrived, many features are just not implementable. This proposal will build on the work of a GSoC project from last year (Parrot-Instrument) and redesign a Parrot debugger that is much more flexible and featureful. Title: Python3 on Parrot Student: Lucian Branescu Mihaila Mentor: Allison Randal This is a very interesting proposal to make PyPy and Parrot place nice and have a mostly self-hosted Python 3 implementation on Parrot VM. Title: Migrating PAST to NQP and newPOST Student: Brian Gernhard Mentor: Bacek This proposal is about improving how Parrot compiler internals work, by rewriting PAST (which is currently written in PIR) in NQP, a higher-level language, as well as migrating to a new version of Parrot Opcode Syntax Tree (POST). Title: A Look-Ahead(1) Left-to-Right Parser Generator for Parrot Student: Tyler Curtis Mentor: darbelo Title: Parrot bindings for GNU Multiprecision (GMP) integer functions Student: Bob Kuo Mentor: Jonathan "Duke" Leto This project will add GMP integer function bindings to Parrot, which will allow all dynamic languages running on Parrot access to these functions! This is a huge savings in terms of obviating the need for every dynamic language needing to create a binding. Having GMP integer function bindings is also a necessary step in implementing native Parrot cryptography and number theory libraries. Title: Javascript on Parrot (code-named Corella) Student: Rohit Jangid Mentor: Will "Coke" Coleda A self-hosted Javascript on Parrot. Can you imagine it? This proposal aims to make it a reality. It builds upon many already-existing Javascript libraries.
